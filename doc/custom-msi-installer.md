# Полное описание этапов кастомного MSI установщика Prizrak-Box

Ниже описана последовательность диалогов и технических действий нового **кастомного MSI**, который собирается напрямую через WiX Toolset (без maker-wix от electron-forge) и используется в GitHub Actions.

## 🚀 Этап 1: Запуск установщика
**Что видит пользователь:**

- Двойной клик по `windows-{arch}.msi`.
- Запрос прав администратора (UAC).

**Что происходит технически:**

- Windows Installer валидирует пакет MSI.
- Загружается UI WiX.
- MSI запускается в per-machine режиме.

## 🌐 Этап 2: Выбор языка
**Диалог:** системный выбор языка Windows Installer (перед стартом UI).

**Что видит пользователь:**

- Окно выбора языка, если доступно несколько языков.
- Варианты: **English** и **Русский**.

**Что происходит технически:**

- В MSI встроены две локализации (`en-us`, `ru-ru`).
- По умолчанию выбирается язык системы.
- Пользователь может вручную переключить язык.

## 👋 Этап 3: Приветствие
**Диалог:** Welcome.

**Что видит пользователь:**

- «Welcome to Prizrak-Box Setup Wizard».

**Что происходит технически:**

- Устанавливаются базовые свойства MSI.
- Подготовка к выбору папки и лицензии.

## 📜 Этап 4: Лицензионное соглашение
**Диалог:** License Agreement (GPLv3).

**Что видит пользователь:**

- Полный текст GPL3.
- Чекбокс «I accept the terms...».

**Что происходит технически:**

- В MSI включены RTF файлы лицензии.
- Кнопка «Next» активна только при согласии.

## 🎯 Этап 5: Выбор компонентов (Feature Tree)
**Диалог:** Custom Setup.

**Что видит пользователь:**

- **Main Application** (обязательный компонент).
- **TUN Service Mode** (опциональный, включен по умолчанию).
- Выбор папки установки.

**Что происходит технически:**

- `Main Application` помечен как обязательный.
- `TUN Service Mode` управляет установкой службы.

## ✅ Этап 6: Подтверждение установки
**Диалог:** Ready to Install.

**Что видит пользователь:**

- Итоговый список выбранных компонентов.
- Кнопка «Install» для запуска установки.

## 🛑 Этап 7: Остановка процессов и служб
**Диалог:** Progress (текст действия).

**Что видит пользователь:**

- Сообщение «Stopping Prizrak-Box processes and services…».

**Что происходит технически:**

- Останавливаются процессы:
  - `Prizrak-Box.exe`
  - `resources\px.exe`
  - `resources\px-service.exe`
- Останавливается и удаляется служба `PrizrakBoxService`:
  - `sc stop PrizrakBoxService`
  - `sc delete PrizrakBoxService`

## 📦 Этап 8: Установка файлов
**Диалог:** Progress.

**Что видит пользователь:**

- Копирование файлов приложения.

**Что происходит технически:**

- Создается папка `C:\Program Files\Prizrak-Box`.
- Копируются файлы приложения и ресурсов.
- Создаются ярлыки и запись протокола `prizrak-box://`.

## ⚙️ Этап 9: Установка службы TUN (опционально)
**Диалог:** Progress.

**Что видит пользователь:**

- Сообщение «Installing Prizrak-Box TUN service…».

**Что происходит технически:**

- Если выбран `TUN Service Mode`, выполняется:
  - `px-service.exe -install`
- Служба регистрируется в системе, без автозапуска.

## 🎉 Этап 10: Завершение установки
**Диалог:** Installation Complete.

**Что видит пользователь:**

- Сообщение об успешной установке.

**Что происходит технически:**

- MSI завершает транзакцию установки.
- Продукт регистрируется в «Программы и компоненты».

---

# 🗑️ Процесс удаления (Uninstall)

## Этап 1: Остановка процессов и служб
- Останавливаются процессы `Prizrak-Box.exe`, `px.exe`, `px-service.exe`.
- Останавливается и удаляется `PrizrakBoxService`.

## Этап 2: Удаление файлов
- Удаляется каталог установки и ярлыки.

## Этап 3: Очистка реестра
- Удаляется регистрация `prizrak-box://`.
- Удаляются ключи `HKLM\Software\legiz-ru\Prizrak-Box`.

---

# 🔄 Процесс обновления (Upgrade)

- `MajorUpgrade` автоматически удаляет старую версию.
- Повторно выполняются остановка процессов и установка службы.
- Позволяется переустановка той же версии (`AllowSameVersionUpgrades="yes"`).
