## Overview

**Prizrak Box** is an open-source, cross-platform proxy client built on top of [Mihomo](https://github.com/MetaCubeX/mihomo) (formerly Clash.Meta).
It provides a user-friendly graphical interface for managing proxy subscriptions, monitoring traffic, and applying routing rules.

![Prizrak Box main screen](../../assets/about-main.webp)

## Key Features

<CardGrid>
  <Card title="Remote Subscriptions" icon="open-book">
    Add and manage remote proxy subscriptions with automatic periodic updates.
    Supports Clash YAML format.
  </Card>
  <Card title="Traffic Monitoring" icon="bars">
    Real-time display of upload/download usage and remaining traffic quota from `Subscription-Userinfo` headers.
  </Card>
  <Card title="Routing Rules" icon="list-format">
    Built-in rule sets (geo-IP, domain lists) with DIRECT, PROXY, and REJECT routing.
    Bundled `.mrs` rule-set archive for offline use.
  </Card>
  <Card title="TUN Mode" icon="laptop">
    System-level traffic capture via TUN interface — routes all device traffic through the proxy without per-app configuration.
  </Card>
  <Card title="Templates (PXD)" icon="puzzle">
    Advanced subscription templates via `Pxd-Template` / `Pxd-Template-Scheme` headers for custom proxy-provider configurations.
  </Card>
  <Card title="Age Encryption" icon="approve-check-circle">
    Supports age-encrypted subscription files. Decrypt with your private key stored securely in the profile.
  </Card>
</CardGrid>

## Subscription Management

![Managing profiles](../../assets/about-profiles.webp)

Prizrak Box supports three types of profiles:

1. **Remote Subscription** — fetched from a URL with optional auto-update interval
2. **Local Configuration** — a local Clash YAML file
3. **Crawl & Merge** — aggregated from multiple sources

When a remote subscription is fetched, the server can return HTTP response headers to control the client behavior — see [Supported Headers](/for-devs/supported-headers/) for the full reference.

## Architecture

The app is structured as:

- **Electron shell** (Windows/macOS/Linux) or **Android activity** — renders the Vue 3 UI
- **Go backend** — handles HTTP API, subscription management, BoltDB persistence
- **Mihomo core** — proxy routing engine embedded in the Go binary
